# Trust system

The crux of this app is *trust*. This has to be maintained at multiple levels.

First is during onboarding. The most obvious first step is to restrict accounts to a single Microsoft tenant. On top of that, mandatory profile picture taken on device?

While searching for a ride, you can choose to only ride with women. You can also make additional choices such as going with quiet passengers (expand on this somehow? what other choices are there).

Review system has to be deeper than star ratings. Common issues (being noisy/inconsiderate, not paying fare, etc) should be directly addressable with a button. However, public facing reviews should be mostly positive/neutral - negative feedback should be addressed as a moderation concern.

Example:
- Public review: this passenger was talkative. (this is not necessarily a negative, but still a preference)
- Private feedback: this passenger made inappropriate comments (actual safety issue, addressed at a platform level)

Nobody is going to read through lots of text reviews so public-facing info should be summarized.

Moderation action can be decided on a case by case basis. We may err on the side of caution and offer a temporary ban until case is reviewed by a human. This is expected to be minimal though considering the audience restriction.
